# Deploy Tiny Tiny RSS Service

[Tiny Tiny RSS](https://tt-rss.org/) 是一個免費且開源的基於網絡的新聞提要 (RSS/Atom) 閱讀器和聚合器

這篇文章將教你如何通過服務市場在 Zeabur 上部署自己的 Tiny Tiny RSS 服務，並綁定自己的域名。

## 步驟一：建立項目

第一步，我們需要建立一個項目。進入 [項目](https://dash.zeabur.com/projects) 頁面，你會看到一個項目列表。

點擊列表末尾的 "創建項目" 按鈕以創建一個新項目。

![Create Project](/deploy/common/create-project.en-US.png)

## 步驟二：部署 PostgreSQL 服務

雖然 Tiny Tiny RSS 兼容 MySQL，但是強烈推薦使用 PostgreSQL 作為數據庫。

首先在服務市場中一鍵部署 PostgreSQL 服務。

按下 "cmd + k" 或者點擊 "添加服務" 按鈕，然後選擇 "服務市場"。

![Select Marketplace](/deploy/common/select-service.en-US.png)

選擇 PostgreSQL

![Deploy PostgreSQL](/marketplace/postgresql/deploy.png)

## 步驟三：部署 Tiny Tiny RSS 服務

你可以在服務市場中一鍵部署 Tiny Tiny RSS 服務。

按下 "cmd + k" 或者點擊 "添加服務" 按鈕，然後選擇 "服務市場"。

![Select Marketplace](/deploy/common/select-service.en-US.png)

在列表中找到 Tiny Tiny RSS 並點擊它以開始部署。

## 步驟四：綁定域名

在 Zeabur 控制台中 Tiny Tiny RSS 服務的 "域名" 塊中，你可以綁定一個域名。

這個域名可以是你自己的域名，也可以是 Zeabur 提供的免費域名。

關於如何為你的服務綁定域名，請參閱 [綁定域名](../deploy/domain-binding)。

## 步驟四：部署 Mercury Parser API 服務（可選）

如果你想要使用 Mercury 插件，你需要額外部署一個 Mercury Parser API 服務。

首先從 Github Fork 一份 [Mercury Parser API](https://github.com/HenryQW/mercury-parser-api) 倉庫

然後新建一個服務，選擇從 Git 構建

![Deploy](/deploy/common/select-service.en-US.png)

還需要在 Zeabur 控制台中 Mercury Parser API 服務的 "域名" 塊中，你需要綁定一個域名獲證生成一個 Zeabur 提供的免費域名。

最後在 Tiny Tiny RSS 中啟用 Mercury 插件，並輸入你的域名。

## 步驟五：開始使用

當部署完成後，你可以通過綁定的域名訪問你的 Tiny Tiny RSS 服務。
