# 網域綁定

**網域**是你的服務的唯一識別碼，它可以用來從網路上造訪你的服務。例如，這個網站的網域是 `zeabur.com`。

在你的服務成功部署到 Zeabur 之後，你可能需要綁定一個網域來將這個服務暴露到網路上。

## 步驟一：打開網域設定

選擇你想要為其加入網域的服務，展開 **網域** 頁籤，你會看到兩個按鈕，用於為你的服務建立網域。

![buttons](/deploy/domain/buttons.png)

## 步驟二：建立網域

Zeabur 提供了一個簡單的服務，用於生成帶有 `zeabur.app` 後綴的網域，這些網域可以用於造訪你的服務，無需額外的配置。

你只需要點選 **生成網域** 按鈕，然後輸入你想要使用的網域，如果這個網域可用且沒有被其他人使用，你就可以點選 **儲存** 按鈕來使用它來造訪你的服務。

![generate](/deploy/domain/generate.png)

你也可以選擇完全自訂你的網域。但是，為了使用你自己的網域造訪你的服務，你需要按照 **步驟三** 中的說明對其進行一些配置。

![custom](/deploy/domain/custom.png)

## 步驟三：設定自訂網域

為了更好的標誌性和靈活性，你可能想要選擇綁定你自己的網域。

如果你還沒有自己的網域，你可以從一些網域服務提供商那裡購買，例如 [GoDaddy](https://www.godaddy.com) 或者 [Cloudflare](https://www.cloudflare.com)。

點選 **自訂網域** 按鈕，輸入你的網域，然後點選 **儲存** 按鈕。

當你將你的網域加入到服務中後，你會發現它處於 `Invalid DNS` 狀態。為了正確造訪你的服務，你需要去你的網域註冊商那裡手動為你的網域配置 DNS 記錄。

![domain](/deploy/domain/invalid.png)

### 配置 DNS 記錄

我們提供了一個使用 Cloudflare 為網域加入 DNS 記錄的例子。

打開 Cloudflare 控制台，選擇你想要配置的網域，然後選擇 DNS 記錄。

![cloudflare](/deploy/domain/cloudflare.png)

在這一頁中，在 **管理 DNS** 的部分，點選 **加入記錄** 按鈕。

填寫相應的名稱和目標內容。這裡，名稱對應於名稱欄位，目標對應於目標欄位。在名稱欄位填寫你的子網域，如果是根網域則填入 **@**，在目標欄位中填入 `cname.zeabur-dns.com`。

注意，這裡應該選擇 CNAME 類型而不是 A 類型。

<Callout>
  如果你之前將 DNS 記錄配置為 `zeabur.app`，你現在必須將其更改為
  `cname.zeabur-dns.com`。`zeabur.app` 將很快被棄用。
</Callout>
