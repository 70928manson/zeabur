# 基本介紹

**Zeabur** 是一個可以幫助你部署服務的平台，而且只需要透過幾個簡單的按鈕即可完成，無論你使用什麼程式語言或開發框架。

你也可以加入我們的 [Discord 伺服器](https://discord.gg/DrdGCvXEyY) 以了解更多以及獲得優先支援！

### Zeabur 的優勢

#### 方便快捷

對比傳統的部署方式，Zeabur 可以讓你在幾分鐘內完成部署，而不需要花費大量的時間去學習部署的相關知識，把時間精力專注於產品的開發。

- 開箱即用的 CI/CD
- 自動識別專案類型
- 快速生成網域
- 多環境隔離

#### 按量計費

Zeabur 採用按量計費的方式，你只需要為你的服務實際用到的資源付費，而不需要為整個伺服器付費。

你可以到[定價文件](/zh-TW/billing/pricing)中了解更詳細的內容。

#### 一站式整合

Zeabur 提供了強大的一站式整合功能。

你可以在一個專案中同時部署前端、後端、資料庫、消息佇列、物件儲存等各種服務，而不需要分別部署到不同的平台。

### 支援的框架和語言

#### Node.js

所有 Node.js 專案皆可以部署，但您需要在 `package.json` 的 `scripts` 欄位加入 `build` 和 `start` 指令。

<Callout emoji="✅">
  如果你的專案是基於現成的框架（例如下方列出的各種框架），則他們已經自動設定好了 `build` 和 `start` 指令，你不需要再自己設定。
</Callout>

特別地，以下框架可以被 Zeabur 自動識別並進行相關最佳化：

- [Next.js](https://nextjs.org/)
- [create-react-app](https://create-react-app.dev/)
- [Vite](https://vitejs.dev/)
- [Remix](https://remix.run/)
- [Umi.js](https://umijs.org/)
- [Nuxt.js](https://nuxtjs.org/)
- [vue-cli](https://cli.vuejs.org/)
- [Qwik](https://qwik.builder.io/)
- [NestJS](https://nestjs.com/)

目前支援的套件管理工具有：

- [npm](https://www.npmjs.com/)
- [yarn](https://yarnpkg.com/)
- [pnpm](https://pnpm.js.org/)

#### Java

- [Spring Boot](https://spring.io/projects/spring-boot)

#### Go

所有基於 [Go Modules](https://blog.golang.org/using-go-modules) 的專案都可以部署。但您需要在專案根目錄有一個 `main.go` 作為程式編譯入口。

如果您有多個程式編譯入口，您需要將他們放在 `cmd` 目錄下，例如一個 `cmd/server-a/main.go` 和一個 `cmd/server-b/main.go`。

Zeabur 會自動識別並部署 `cmd` 目錄下與 Service 名稱相同的程式編譯入口。

#### Python

如果您的專案根目錄有 `main.py` 或 `app.py`，Zeabur 會自動以 Python 專案的方式進行部署。

您需要將專案需要的外部套件寫入 `requirements.txt`, `Pipfile` 或 `Pipfile.lock` 檔案，讓 Zeabur 能夠在建置以前自動安裝。

特別地，以下框架可以被 Zeabur 自動識別並進行相關最佳化：

- [Django](https://www.djangoproject.com/)
- [Flask](https://flask.palletsprojects.com/)

如果您要部署 Django 專案，請確保您的專案根目錄有 `manage.py` 且至少一個子目錄有 `wsgi.py`。

#### Deno

如果您的專案根目錄有 `deno.json` 或 `fresh.gen.ts`，Zeabur 會自動以 Deno 專案的方式進行部署。

您需要在 `deno.json` 的 `tasks` 欄位加入 `start` 指令。

如果您的專案是基於現成的Fresh框架，則他們已經自動設定好了`start` 指令，您不需要再自己設定。

- [Fresh](https://fresh.deno.dev/)

#### 靜態網站

所有靜態網站都可以部署，但您需要在專案根目錄有一個 `index.html` 作為網站入口。

特別地，Zeabur 能夠識別出以下的靜態網站生成框架所生成的靜態網站並進行相關最佳化：

- [Hugo](https://gohugo.io/)
- [Hexo](https://hexo.io/)

#### MongoDB

您可以透過 Zeabur 部署一個 MongoDB 服務，並且可以透過 Zeabur 自動注入到同個環境中其他服務的環境變數來讓您的其他服務連接到 MongoDB 資料庫。
