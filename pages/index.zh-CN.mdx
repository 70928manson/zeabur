# 基本介绍

**Zeabur** 是一个可以帮助你部署服务的平台，无论你使用什么编程语言或开发框架，你都只需要通过几个简单的按钮进行部署。

### Zeabur 的优势

#### 方便快捷

对比传统的部署方式，Zeabur 可以让你在几分钟内完成部署，而不需要花费大量的时间去学习部署的相关知识，把时间精力专注于产品的开发。

- 开箱即用的 CI/CD 
- 自动识别项目类型
- 快速生成域名
- 多环境隔离

#### 按量计费

Zeabur 采用按量计费的方式，你只需要为你的服务实际用到的资源付费，而不需要为整个服务器付费。

你可以到[定价文档](/zh-CN/billing/pricing)中了解更详细的内容。

#### 一站式整合

Zeabur 提供了强大的一站式整合功能。

你可以在一个项目中同时部署前端、后端、数据库、消息队列、对象存储等各种服务，而不需要分别部署到不同的平台。

### 支持的框架和语言

#### Node.js

所有 Node.js 项目皆可以部署，但您需要在 `package.json` 的 `scripts` 栏位中加入 `build` 和 `start` 命令。

<Callout emoji="✅">
  如果你的项目是基于现成的框架（例如下方列出的各种框架），则他们已经自动设定好了 `build` 和 `start` 指令，你不需要再自己设定。
</Callout>

特别地，以下框架可以被 Zeabur 自动识别并进行相关优化：

- [Next.js](https://nextjs.org/)
- [create-react-app](https://create-react-app.dev/)
- [Vite](https://vitejs.dev/)
- [Remix](https://remix.run/)
- [Umi.js](https://umijs.org/)
- [Nuxt.js](https://nuxtjs.org/)
- [vue-cli](https://cli.vuejs.org/)
- [Qwik](https://qwik.builder.io/)
- [NestJS](https://nestjs.com/)

目前支持的包管理工具有：

- [npm](https://www.npmjs.com/)
- [yarn](https://yarnpkg.com/)
- [pnpm](https://pnpm.js.org/)


#### Java

- [Spring Boot](https://spring.io/projects/spring-boot)

#### Go

所有基于 [Go Modules](https://blog.golang.org/using-go-modules) 的项目都可以部署。 但您需要在项目根目录有一个 `main.go` 作为程序编译入口。

如果您有多个程序编译入口，则需要将他们放在 `cmd` 目录下，例如一个 `cmd/server-a/main.go` 和一个 `cmd/server-b/main.go`。

Zeabur 会自动识别并部署 `cmd` 目录下与 Service 名称相同的程序编译入口。

#### Python

如果您的项目根目录有 `main.py` 或 `app.py`，Zeabur 会自动以 Python 项目的方式进行部署。

您需要将项目需要的外部套件写入 `requirements.txt`, `Pipfile` 或 `Pipfile.lock` 文件，让 Zeabur 能够在构建以前自动安装。

特别地，以下框架可以被 Zeabur 自动识别并进行相关优化：

- [Django](https://www.djangoproject.com/)
- [Flask](https://flask.palletsprojects.com/)

如果您要部署 Django 项目，请确保您的项目根目录有 `manage.py` 且至少一个子目录有 `wsgi.py`。

#### Deno

如果您的项目根目录有 `deno.json` 或 `fresh.gen.ts`，Zeabur 会自动以 Deno 项目的方式进行部署。

您需要在 `deno.json` 的 `tasks` 栏位中加入 `start` 命令。

如果你的项目是基于现成的Fresh框架，则他们已经自动设定好了 `start` 指令，你不需要再自己设定。

- [Fresh](https://fresh.deno.dev/)

#### 静态站点

所有静态站点都可以部署，但您需要在项目根目录有一个 `index.html` 作为网站入口。

特别地，Zeabur 能够识别出以下的静态站点生成框架所生成的静态站点并进行相关优化：

- [Hugo](https://gohugo.io/)
- [Hexo](https://hexo.io/)

#### MongoDB

您可以通过 Zeabur 部署一个 MongoDB 服务，并且可以通过 Zeabur 自动注入到同个环境中其他服务的环境变量来让您的其他服务连接到 MongoDB 数据库。
