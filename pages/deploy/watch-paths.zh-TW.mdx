# 觸發路徑

在服務部署完成後，您可能經常會更新程式碼，但有些程式碼更新並不需要讓某個服務進行重新部署。

例如，在一個將前後端程式碼放在同一個 Git 儲存庫的專案中，也許您只是更新了後端的部分程式碼，但提交以後，發現前端的服務也一起重新部署了。

想要避免這一點，仅需前往設定修改觸發路徑：

### 第一步：打開服務設定頁面

點擊想修改配置的服務，並選擇設定頁面，您將會看到一個輸入框 **Watch paths** ，供您修改服務的觸發路徑。

![setting](/deploy/watch-path/setting.png)

### 第二步：修改觸發路徑

您將會看到，預設的觸發路徑為 `*`，代表服務觸發整個儲存庫，儲存庫內任何檔案的更新都會帶來服務的重新部署。

您可以按照 `.gitignore` 檔案的格式一樣，修改自己的觸發路徑。

唯一不同的是， `.gitignore` 代表的是忽略，而此處代表的是觸發。

![repo](/deploy/watch-path/repo.png)

而您為這個儲存庫部署了兩個服務，一個服務的根目錄設定為了 `client` 一個為 `server`，但這樣只是為其指明了建置的路徑，要想做進一步的隔離，您就需要去設定 **Watch paths**。

例如，將客戶端的服務的 **Watch paths** 設定為 `/client` 之後，您在 `server` 資料夾中做的任何更新都不會自動觸發客戶端服務的重新部署了。
